Sinu,

Still a fair bit to go. The idea sounds simple but they take time to get 
right.

Here I'm just trying to get the full circle of price-relaying working. 
And make it very simple to use.

What I've ended up doing is using DWSII (Delphi web script) to do the 
actual price calculations. This means that the users can put in whatever 
logic that they like to calculate from the cost price up to the final 
sell price.

The Relay_Calculations.pas file sits in the PreisShare directory. It 
get's interpreted, not compiled into the program.

I've attached a sample.

An Editor (with syntax highlighting) is built into the new PriceSharer. 
c/o the SynEdit components.

We've got this running in production now and it seems to be ok.

So it will be nice to be able to download some prices from an email, run 
the relay_calculations over the prices, and then pump out new prices to 
customers. We're getting ever closer.
